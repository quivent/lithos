2. Memory Map on GH200

GH200 provides unified virtual address space across HBM3e and LPDDR5X. The NVLink-C2C bridge makes both visible to GPU kernels. Lithos uses mmap to place safetensors files directly in this space.

4. One Token's Journey

Step by step through 64 layers. Each layer runs exactly 2 kernel launches: one for the sequence mixer (DeltaNet or Attention) and one for the MLP.

embed EMBED token ID → EMBED kernel → activation [1, 4096]
layer 0 DeltaNet input → RMSNorm → conv1d → recurrence → out_proj → residual → MLP → out
layer 1 DeltaNet input → RMSNorm → conv1d → recurrence → out_proj → residual → MLP → out
layer 2 DeltaNet input → RMSNorm → conv1d → recurrence → out_proj → residual → MLP → out
layer 3 Attention input → RMSNorm → QKV proj → RoPE → flash attn → out_proj → residual → MLP → out
··· layers 4–62 follow the same pattern: 3 DeltaNet then 1 Attention, repeating ···
layer 63 Attention input → RMSNorm → QKV proj → RoPE → flash attn → out_proj → residual → MLP → out
output HEAD RMSNorm → lm_head GEMV → logits [1, vocab] → SAMPLE (CPU) → token ID → detokenize → text

Key insight: in DeltaNet layers, RMSNorm + conv1d + recurrence + output projection are all fused into the single FUSED-DELTANET kernel launch. Similarly, RMSNorm + QKV + RoPE + attention + output projection fuse into FUSED-ATTENTION. The MLP (gate + up + SiLU + down) fuses into FUSED-MLP. This is why 2 kernel launches per layer, not 10–20.

Every layer in the vLLM stack that is not "Driver" or "GPU" exists to translate between abstractions. Lithos removes the translations.

What Each Removed Layer Costs

Removed Layer Overhead Type Cost
Python (vLLM) GIL contention, object allocation, scheduler latency 50–200 µs/token
Python (PyTorch) Tensor creation, dispatch, reference counting 20–100 µs/op
ATen dispatcher Virtual dispatch, dtype/device checks, schema matching 5–15 µs/op
cuBLAS Heuristic selection, workspace alloc, generic kernels 10–50 µs/call
CUDA Runtime cudaMalloc pool, stream sync, error checking 5–30 µs/call

At batch=1, these overheads dominate: 3,200 kernel launches × 40 µs average overhead = 128 ms of pure waste per token. Lithos has 129 launches × ~0 overhead = the GPU does real work.
