Language Reference

Two targets. One language. Every primitive maps to one instruction on the target.

Overview

Lithos source files use the .ls extension. The language has two targets (ARM64 host code and Hopper sm90a GPU code) but one surface syntax. Every primitive maps to exactly one target instruction. There are no functions, no call/return. Compositions are named sequences that the compiler flattens into instruction streams at compile time.

Indentation delimits composition bodies. Tokens are whitespace-separated. Unicode symbols serve as primitive names — the lexer recognises them by their UTF-8 byte sequences.

Examples

RMSNorm — normalize a vector by its root mean square

rmsnorm x w D :
    each i
        sq → 32 x i * x i
        s Σ sq
        rms √ s / D
        ← 32 out i rms * w i

Sigmoid — 1 / (1 + e^(-x))

sigmoid x :
    neg x * -1
    ex e^ neg
    s 1 + ex
    1/ s

SiLU — x · sigmoid(x)

silu x :
    sig sigmoid x
    x * sig

L2 normalize

l2norm x :
    sq x ** x
    s Σ sq
    irs 1/√ s
    x ** irs

Token sampling — argmax of logits

sample logits :
    # △ logits

Syscall — open a file

open path flags mode :
    ↓ $8 56
    ↓ $0 -100
    ↓ $1 path
    ↓ $2 flags
    ↓ $3 mode
    trap
    ↑ $0

MMIO — poke a GPU register via BAR0

gsp_poke bar0 offset val :
    ← 32 bar0 + offset val

DeltaNet layer — composition of compositions

deltanet_layer x X :
    rmsnorm x w_norm D
    project W_q x q
    project W_k x k
    project W_v x v
    l2norm k
    silu q
    l2norm q
    matvec S k recall
    ← 32 out recall

Architecture Register Dictionaries

Each target has a dictionary mapping names to hardware register IDs. $N is a numbered slot. $NAME looks up the dictionary.

arch/hopper.dict

TID_X       S2R  0x21
TID_Y       S2R  0x22
CTAID_X     S2R  0x25
LANEID      S2R  0x00

arch/arm64.dict

CNTVCT_EL0  MRS  0x5F01
MPIDR_EL1   MRS  0xC005

The compiler dispatches to the right dictionary based on the surrounding composition's target. GPU compositions see Hopper registers; host compositions see ARM64 system registers.
