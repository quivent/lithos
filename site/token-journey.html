What Happens to a Token

One token through 64 layers of Qwen 3.5-27B. Every transformation, explained.

Before the Layers

The word "Paris" arrives as text. The tokenizer converts it to a number — say token ID 12037. That number is an index into a table of 248,320 rows (one per word or word-piece the model knows). Row 12037 is a list of 5,120 numbers. These numbers are the model's learned representation of "Paris" — a vector.

This vector is ALL the model knows about "Paris" at this point. 5,120 numbers. It will be transformed 64 times (once per layer) before the model decides what comes next.

One Full Attention Layer

Qwen 3.5-27B has 64 layers. Of those, 16 are full attention layers (layers 3, 7, 11, 15, ... — every 4th layer). Here is what happens to the 5,120-number vector in one of those layers, step by step.

a) Normalize

The numbers get rescaled so their average magnitude is 1. This prevents the values from drifting too large or too small as they pass through layers. The formula: divide each number by the root-mean-square of all 5,120 numbers, then multiply by a learned weight (5,120 learned weights, one per position). This operation is called RMSNorm (Root Mean Square Normalization). Output: still 5,120 numbers, but balanced.

b) Project to Q, K, V

The normalized vector gets multiplied by three separate weight matrices to produce three new vectors:

  • Q (query): "what am I looking for?" — 24 heads × 256 numbers = 6,144 numbers
  • K (key): "what do I contain?" — 4 heads × 256 numbers = 1,024 numbers
  • V (value): "what information do I carry?" — 4 heads × 256 numbers = 1,024 numbers

Each multiplication is: take the 5,120-number input, multiply by a matrix of learned weights, get a different-sized output. The weights are the model's knowledge — billions of numbers learned during training. This is a matrix multiplication (often called a "projection" because it projects the vector into a new space).

Qwen 3.5 uses Grouped Query Attention: 24 query heads share 4 key-value heads (each key-value head serves 6 query heads). This saves memory without losing much quality.

c) Rotate (RoPE)

The Q and K vectors get rotated by angles that depend on the token's position in the sequence. This technique is called Rotary Position Embedding (RoPE). Token 1 gets small rotations. Token 100 gets larger rotations. This is how the model knows word ORDER — "the cat sat" versus "sat the cat" produce different rotations, so the model can distinguish them even though the same words appear.

d) Store K and V in the cache

The K and V vectors for this token get appended to the KV cache — a growing table of every previous token's K and V. This is the model's memory of the conversation so far. Every time a new token arrives, its K and V get added to the table. The table only grows; it is never recomputed.

e) Attention

This is the core operation. The Q vector (what am I looking for?) gets compared against every previous token's K vector (what does each past token contain?). Each comparison produces a score — how relevant is that past token to the current one.

The scores get normalized using softmax (a function that converts a list of arbitrary numbers into probabilities that sum to 1). Then each past token's V vector gets weighted by its score and summed. The result: a 6,144-number vector that represents "the relevant information from all past tokens, as seen by this token."

The 6,144-number attention result gets multiplied by another weight matrix to produce a 5,120-number vector. This projects the multi-head result back into the model's standard vector size.

The output gets ADDED to the original input (before normalization). The MLP (Every Layer, After Attention or DeltaNet)

Every one of the 64 layers has a Multi-Layer Perceptron (MLP) that runs after the attention (or DeltaNet) step. The MLP transforms each token's vector independently — no interaction between tokens here, only within one token's own representation.

h) Normalize again

Same RMSNorm as step (a). The vector is rescaled to balanced magnitude before entering the MLP.

i) Gate and Up projections

The vector gets multiplied by TWO separate weight matrices simultaneously:

  • Gate: 5,120 → 17,408 numbers
  • Up: 5,120 → 17,408 numbers

The vector expands from 5,120 to 17,408 numbers. This expansion gives the MLP a wider space to work in — more room to represent complex patterns before compressing back down.

j) Activate

The gate output goes through SiLU (Sigmoid Linear Unit) — a smooth curve that lets positive values pass through nearly unchanged while smoothly zeroing out negative values. Then the SiLU output gets multiplied element-by-element with the up output.

This is the nonlinearity — the step that makes deep networks powerful. Without it, 64 layers of matrix multiplication would mathematically collapse into a single multiplication. The nonlinearity is what allows each layer to learn something the previous layers could not express.

k) Down projection

The 17,408-number result gets multiplied by a weight matrix back to 5,120 numbers. The wide representation is compressed back to the model's standard vector size.

l) Add residual again

Add the MLP output to the input (before this MLP's normalization). Same skip connection as step (g).

One DeltaNet Layer

DeltaNet layers replace attention (steps b-f) with a different mechanism. The MLP (steps h-l) is the same.

Instead of comparing against every past token (which takes longer as the conversation grows), DeltaNet maintains a compressed state matrix — 16 heads × 128 × 128 = 262,144 numbers per layer. This state is a compressed summary of all past tokens.

48 of the 64 layers use DeltaNet (layers 0, 1, 2, 4, 5, 6, 8, 9, 10, ... — every layer that is NOT every 4th).

DeltaNet steps (replacing attention)

a) Normalize

Same RMSNorm as the attention layer.

b) Project to Q, K, V, gate, beta

The normalized vector gets multiplied by weight matrices to produce Q, K, V vectors plus two additional outputs: a gate (controls how much new information to write) and beta (a per-head scaling factor). Same kind of matrix multiplication as attention, different output sizes.

c) Convolution

A small 4-tap filter (Conv1D) runs over the K vector and the last 3 K vectors (stored in a tiny buffer). This gives the model a short-term memory of very recent tokens — a simple pattern detector that catches local context before the recurrence processes it.

d) State update

This is the core DeltaNet operation:

  • Read from state: multiply K against the state matrix to retrieve what the state "remembers" about this key pattern
  • Compute the difference between V and what was retrieved
  • Write to state: add the new information (scaled by gate and beta) to the state matrix

The state matrix gets slightly modified by every token. It is a running summary that gets updated, not a growing list like the KV cache.

e) Query the state

Multiply Q against the (just-updated) state matrix to get the output. This replaces the attention scores + weighted sum — instead of scanning all past tokens, it reads from the compressed state.

f) Project output and add residual

Same as attention: multiply by a weight matrix to get back to 5,120 numbers, then add the residual connection.

After the DeltaNet steps, the same MLP (steps h-l) runs. Every layer — whether attention or DeltaNet — has a sequence mechanism followed by an MLP.

Why hybrid?

DeltaNet is fast: same cost per token regardless of how long the conversation is. The state matrix is a fixed size (262,144 numbers per layer), so processing the 10,000th token costs the same as processing the 10th. But DeltaNet is imprecise — the state compression loses detail. Information from thousands of tokens gets squeezed into a fixed-size matrix.

Full attention is expensive: cost grows with conversation length (every new token must compare against every past token). But full attention is exact — every past token is considered individually, nothing is compressed away.

DeltaNet (48 layers)

Fixed cost per token. State: 128 × 128 matrix per head. Fast but lossy — compresses all history into fixed-size state.

Full Attention (16 layers)

Cost grows with sequence length. KV cache stores every past token. Exact but expensive — scans all history every time.

Qwen 3.5 uses DeltaNet for 3 out of every 4 layers, then full attention every 4th layer to correct any drift. Cheap processing for most of the work, with occasional precise correction. The 16 attention layers act as checkpoints that re-anchor the representation against exact history.

After All 64 Layers

The token's 5,120-number vector has been transformed 64 times. It no longer represents "Paris" — it represents "what should come after 'The capital of France is Paris' given everything the model knows."

One final step: a matrix multiplication of the 5,120-number vector against the embedding table (transposed): 5,120 × 248,320 → one score per vocabulary word. The highest score wins. Maybe it is "." or "," or " and" — whatever the model predicts comes next.

This process repeats for every token the model generates. Each new token goes through all 64 layers, reads from and writes to the caches and states, and produces one more prediction. One token at a time, thousands of times per second.

The KV cache grows with every token — a 10,000-token conversation adds ~640 MB to the cache. The DeltaNet state stays at 12.6 MB regardless of conversation length. This is the fundamental trade-off: attention is precise but expensive; DeltaNet is compact but approximate.
